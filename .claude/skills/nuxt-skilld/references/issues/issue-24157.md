# add global interceptors for `useFetch` without creating `customUseFetch`

### Describe the feature

# Description:
The proposal is to implement request and response interceptors for the `useFetch` composable in Nuxt 3, similar to how Axios interceptors function. These interceptors would allow users to globally manage, modify, or intercept HTTP requests before they are sent and responses before they are handled.

# Use Case:
Integrating request and response interceptors within the `useFetch` composable would be highly beneficial for various scenarios. For instance, users could add headers, perform pre-processing or transformations on outgoing requests, log requests or responses, handle errors, or modify responses in a uniform and centralized manner across their application.

# Expected Behavior:
The request interceptors would enable users to intercept and modify the outgoing request before it is sent, while the response interceptors would allow users to intercept and modify the response before it is further processed. These interceptors should seamlessly integrate with the existing useFetch functionality, ensuring flexibility and ease of use.

# Examples or Use Scenarios:
```javascript
// Global request interceptor example
onRequest((config) => {
  // Modify the request config before sending
  const token = useCookie('token');
  config.headers['Authorization'] = `Bearer ${token.value}`
  return config
});

// Global response interceptor example
onResponse((response) => {
  // Log the response or modify it before further processing
  console.log('Response:', response)
  return response
});
```

# Potential Impact:
This feature would significantly enhance the flexibility and control for developers using the `useFetch` composable in Nuxt 3. It would streamline the process of globally handling requests and responses, making it easier to manage authentication, logging, error handling, and other common functionalities across the application.
