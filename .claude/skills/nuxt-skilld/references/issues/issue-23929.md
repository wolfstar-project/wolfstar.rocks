### Describe the bug

The problem is when using layout named slots within pages to show different piece of data for each page. For example different menu, toolbar, header info or whatever. On each page/route change, whole layout gets re-executed. So if you have components or subcomponents with fetch calls, everything is refetched every time, event listeners are duplicated and memory usage goes up.

If NuxtLayout is added to app.vue instead pages, everything works fine. But in that case, I am unable to use named slots within pages.
