# Windows HMR breaks with Tailwind v4 and file-based routing in Nuxt v4

### Environment

- Operating System: `Windows_NT`
- Node Version:     `v22.17.0`
- Nuxt Version:     `4.0.0-rc.0`
- CLI Version:      `3.25.1`
- Nitro Version:    `2.11.13`
- Package Manager:  `bun@1.2.18`
- Builder:          `-`
- User Config:      `compatibilityDate`, `devtools`, `css`, `vite`, `app`
- Runtime Modules:  `-`
- Build Modules:    `-`


### Reproduction

I don't see a StackBlitz Nuxt v4 template. If there is one, please let me know.

### Describe the bug

After adding Tailwind v4 to a Nuxt 4 RC0 project, and enabling file based routing, HMR stops working.  Changes to index.vue, for example, are not seen in the browser until a manual refresh in the browser is performed. If I disable Tailwind and/or file based routing, then HMR starts working again. Also, leaving file based routing enabled and only removing Tailwind, HMR also starts working again.  Tailwind seems to be prime factor, but only if FBR is used.

- This happens with both bun and npm
- I have not see this with Nuxt 3.17.x

### Additional context

I don't know if this matters, but with Tailwind and FBR enabled, I see this in the console after making a change to index.vue:
`hmr update /pages/index.vue?macro=true, /@id/virtual:nuxt:D%3A%2FNuxt%2Fnuxt-app%2F.nuxt%2Froutes.mjs`

When Tailwind is disabled, it shows this instead when I make a change to index.vue:
`hmr update /pages/index.vue, /pages/index.vue?macro=true, /@id/virtual:nuxt:D%3A%2FNuxt%2Fnuxt-app%2F.nuxt%2Froutes.mjs`
