# Manually set types for runtimeConfig are overwritten by the inferred types in nuxt.config

### Environment

------------------------------
- Operating System: `Darwin`
- Node Version:     `v18.12.1`
- Nuxt Version:     `3.2.3`
- Nitro Version:    `2.3.3`
- Package Manager:  `pnpm@8.2.0`
- Builder:          `vite`
- User Config:      `ssr`, `app`, `modules`, `typescript`, `runtimeConfig`, `imports`, `vite`, `i18n`, `tailwindcss`
- Runtime Modules:  `@nuxtjs/i18n@8.0.0-beta.10`, `@nuxtjs/tailwindcss@6.6.5`, `@pinia/nuxt@0.4.8`, `@vueuse/nuxt@10.0.0`
- Build Modules:    `-`
------------------------------

### Reproduction

Set some types manually in `index.d.ts` file:
```
declare module 'nuxt/schema' {
    interface PublicRuntimeConfig {
        APP_ENV: 'development' | 'staging' | 'production'
    }
}

export {}
```
Then in `nuxt.config` set this same variable to another type:
```
runtimeConfig: {
    public: {
        APP_ENV: 123, // will infer `number` type and overwrite the manual types
    },
}
```
Using `useRuntimeConfig().public.APP_ENV` will now be typed as a number and there is no type error reported in `nuxt.config`.

### Describe the bug

I'd like to define union type for an env variable in a `env.d.ts` file. If I then also reference this variable in `nuxt.config.ts` under runtimeConfig, that union type is lost and overwritten by the auto-inferred type.
