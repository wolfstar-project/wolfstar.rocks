# Easier way to set up a `src` dir in Nuxt 4

### Describe the feature

In Nuxt 3, setting up a `src` directory was easy. By that I mean a directory which contains "all source code" but no configuration files, node_modules, or nuxt.config.ts. By setting the `srcDir` in your `nuxt.config.ts` one was good to go.

With Nuxt 4, the role of the `srcDir` changed: It not only became `app/` by default but also contains only a part of your source code (e.g. no server, modules or layers folder). To achieve the same behavior as in Nuxt 3, one would need more configuration:


```ts
// In your nuxt.config.ts
export default defineNuxtConfig({
  srcDir: 'src/app',
  serverDir: 'src/server',
  dir: {
    public: 'src/public',
    modules: 'src/modules',
    shared: 'src/shared',
  },
})
```

In addition to that, some parts (e.g. the `layers` folder) are not configurable at the moment. Same applies to other folders coming from third party plugins (e.g. `content` from `nuxt/content`).

Time for your thoughts and feedback!

Curious to hear how many of you used a `src` (or differently named) folder in Nuxt 3 and would like to see an easier integration, either via module or via core config option.
