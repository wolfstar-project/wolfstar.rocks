# Factory function to create `useFetch` and `useAsyncData` with defaults

### Describe the feature

It is common that a (bigger) project and library/module authors want to create a composable that is _based_ on either `useFetch` or `useAsyncData` and provides some defaults.

There is a nice example by @516310460 that demonstrates it in nuxt/framework#6995. 

The basic idea is to create factory composables that wrap `useFetch` and `useAsyncData` with defaults while preserving the behavior:

- `export const useMyAPI = createFetch(/* defaults */)`
- `export const useMyAPI = createAsyncData(/* defaults */)`

Currently it is possible with `$fetch.create` to make `const useMyAPI = $fetch.create(/* defaults */)` but it is just plain fetch without benefits of `useFetch` with reactivity.

With this new factory function, we can also find an opportunity to auto-generate keys for these custom instances too based on eager scanning with AST or fast regex. Currently, if a wrapper is custom-made, the key and state of any usage are shared and the same.

```js

  onRequest({ options }) {
    options.headers = options.headers || {};
    options.headers['X-Test'] = 'foo bar';
  },
});

**@edgarsn** (+9):

In context of this feature, I would like to point out suggestion. We could allow defaults not only as an object, but as a function as well. 

My current implementation is something like this:

```typescript
export function useCustomFetch<ResT = void,
    ErrorT = Error,
    ReqT extends NitroFetchRequest = NitroFetchRequest,
    _ResT = ResT extends void ? FetchResult<ReqT> : ResT,
    Transform extends (res: _ResT) => any = (res: _ResT) => _ResT,
    PickKeys extends KeyOfRes<Transform> = KeyOfRes<Transform>>(
    request: Ref<ReqT> | ReqT | (() => ReqT),
    opts: UseFetchOptions<_ResT, Transform, PickKeys> = {}
) {
    const authToken = useAuthToken().value;

    opts = opts || {};

    let headers: HeadersInit = {
        ...opts?.headers,
    };

    if (authToken && headers['Authorization'] !== null) {
        headers['Authorization'] = 'Bearer ' + authToken;
    }

    opts.headers = headers;
    
    return useFetch<ResT, ErrorT, ReqT, _ResT, Transform, PickKeys>(request, opts);
}
```...
