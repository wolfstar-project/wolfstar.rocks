# Security Vulnerabilities in @prisma/dev dependencies (hono)

### Bug description

`npm audit` reports high severity vulnerabilities in `hono`, which are transitive dependencies of `prisma` via `@prisma/dev`.

### Severity

 Critical: Data loss, app crash, security issue

### Reproduction

1. Install `prisma@7.2.0`
2. Run `npm audit`

### Expected vs. Actual Behavior

**Expected Behavior:**
`npm audit` should report 0 high-severity vulnerabilities when installing `prisma`. The `hono` dependency pulled in by `@prisma/dev` should be updated to a secure version.

Ideally, `@prisma/dev` would not strictly lock versions, making it possible to upgrade without overrides when issues like this and #28752 occur.

**Actual Behavior:**
npm audit reports 2 high-severity vulnerabilities in hono, introduced via @prisma/dev.

### Frequency

Consistently reproducible

### Does this occur in development or production?

Both development and production

### Is this a regression?

No, this is likely due to new vulnerabilities being discovered in the dependencies used by the latest version.

### Workaround

Yes, adding an overrides section to package.json forces the use of secure versions:

```json
"overrides": {
  "hono": "^4.11.4",
}
```
