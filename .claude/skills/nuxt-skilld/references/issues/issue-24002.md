# OpenTelemetry support

### Describe the feature

By default there is no way to add OpenTelemetry (otel) to the Nuxt app. It would be very important to support otel within Nuxt to get even more information of the app when running on any environment. Currently there is only one possible solution to add otel within Nuxt; building and require the instrumentation via Node.js:

```sh
$ npx nuxt build
$ node --require ./instrumentation.js ./output/server.js
```

I am pretty new to Nuxt, but the implementation could be as easy as adding another entrypoint in the webpack/vite files or a little more complex if Nuxt has serverless functions (which I haven't seen just yet). 

Currently I am not sure if a nuxt-module will solve the problem, as it would be instrumented way to late, as far as I have understand the modules.

On the usage side of Next it looks like this, which could be an example of how the usage could look like for Nuxt.

There are existing discussions: https://github.com/nuxt/nuxt/discussions/23409, https://github.com/nuxt/nuxt/discussions/16498 and https://github.com/nuxt/nuxt/discussions/16006, which seem more like a "How to", rather than requesting a new feature.
