# parallel data loading support for `useAsyncData`

At the moment, the Nuxt data fetching utilities like `useAsyncData` and `useFetch` are blocking by default. You can either pick:

1. a data fetching composable must be awaited, and will block _both_ page transitions and other components
2. a data fetching composable is lazy and will block _neither_ page transitions nor other components

There is a scenario in-between, for example:

1. several components or pages need data
2. we don't want each component to block the next component
3. we _do_ want everything to have loaded before we change route

I propose adding an additional `strategy` option to `useAsyncData`, which can take either `lazy`, `parallel`, or `blocking` (or maybe that should be `eager`?). `lazy: true` becomes an alias for `strategy: 'lazy'` so this is a backwards-compatible enhancement.

I have a working POC and would value comments and thoughts before opening a PR.
