# [Discussion] Make auto-imports optional by default

### Describe the feature

This is a follow up from same Nitro issue: https://github.com/nitrojs/nitro/issues/2232
Nitro is going to make auto-imports an opt-in feature. I think all arguments from this issue also applies to Nuxt, but here my summary:

I would love auto imports being fully disabled by default + all the docs to use explicit imports. Personally I see no benefits of having auto imports, but a lots of downsides. E.g. at the moment it is not clear to developers what actually comes from H3, what from Nitro, what from Nuxt and what from Vue. Nowadays imports are added by the IDE automatically anyway, so every dev should already have an "auto imports" experience.

Btw. I failed multiple times disabling auto imports within a Nuxt 3 app, as well as in a Nitro 2 app: https://github.com/nitrojs/nitro/issues/2305

I really hope Nuxt 4 will fully opt-out auto imports, docs will be updated to show imports and Nuxi sets opt-out as a default. Within Nuxt 5, auto-imports could be disabled by default and just be an opt-in.
