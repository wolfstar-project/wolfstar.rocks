# Accessibility roadmap

We might build these directly into nuxt or possibly via a core or adhoc module or even decoupled module like `nuxt/a11y`.

- [x] https://github.com/nuxt/nuxt/issues/14673
- [x] built-in axe core
- [x] devtools integration? cc: @antfu
- [ ] https://github.com/nuxt/nuxt/issues/23375
- [ ] other built-in components or composables, e.g. for announcement?
- [ ] a convenient way to move focus on every navigation
- [ ] skip links for traversing to/from the top-most layout that changed

I just wanted to chime in to say that I love this initiative. It would be great if Nuxt helped with accessible client-side routing, which is is often the biggest omission in full stack frameworks.

Route announcer is a good first step, but is generally not enough and also needs to be possible to disable because it could conflict with focus announcements. Some other things to consider adding:
- a convenient way to move focus on every navigation.
- skip links for traversing to/from the top-most layout that changed.
- documentation and examples on accessible routing patterns.
