# Prisma can't connect to Docker PostgreSQL database P1000 when trying to migrate

### Question

I am getting this error message from prisma after trying ```bunx --bun prisma migrate dev```:
```
Error: P1000: Authentication failed against database server, the provided database credentials for `user` are not valid.
```

and this error message from Docker:
```
2026-02-09 12:31:52 postgres-1  | 2026-02-09 17:31:52.231 UTC [35] FATAL:  password authentication failed for user "user"
```

I am on prisma 7.3 and I can't put the database url in the schema anymore I have to have it in prisma.config.ts
```
// This file was generated by Prisma, and assumes you run Prisma commands using `bun --bun run prisma [command]`.
import { defineConfig, env } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations",
  },
  datasource: {
    url: env("DATABASE_URL"),
  },
});
```...

## Accepted Answer

this is almost always a stale docker volume issue. when postgres initializes for the first time, it writes the credentials to the data directory. if you later change `POSTGRES_USER` or `POSTGRES_PASSWORD` in your `.env`, postgres ignores the new values because the data directory already exists.

### fix

```bash
docker compose -f dev-docker-compose.yml down -v
docker compose -f dev-docker-compose.yml up -d
```

the `-v` flag removes the named volume (`postgres_data`), so postgres reinitializes with whatever credentials are currently in your `.env`.

### verify your credentials match

make sure your `.env` has something like:

```
POSTGRES_USER=user
POSTGRES_PASSWORD=password
POSTGRES_DB=db
POSTGRES_PORT=5432
```

and that your `DATABASE_URL` uses the exact same values:

```
DATABASE_URL=postgresql://user:password@localhost:5432/db
```

if these don't match up, prisma will connect to the container fine (since it's on `localhost:5432`) but postgres will reject the auth.
